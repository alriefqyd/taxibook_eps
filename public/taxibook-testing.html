Test against the production URL, not localhost — push notifications and PWA install require HTTPS.

Pre-Test Setup & Cron Verification
Setup
Do this before anything else. If the cron secret is wrong, auto-complete and reminders will silently fail for all real users.
  • CRON_SECRET set in Vercel Vercel → Project → Settings → Environment Variables → confirm CRON_SECRET exists
  • cron-job.org sends Authorization header Job → Advanced → Request Headers: Authorization: Bearer <same value>
  • Manually call the cron endpoint and get 200 GET https://your-app.vercel.app/api/cron/auto-complete — with Authorization header — response: {"success":true,...}
  • At least 1 active taxi assigned to driver test account in Supabase
  • All 3 role accounts can log in successfully
2
PWA Installation
All roles
  • Android — Add to Home Screen Chrome → ⋮ menu → "Add to Home Screen" → icon shows Vale logo, name shows "TaxiBook EPS"
  • iOS — Add to Home Screen Safari → Share → "Add to Home Screen" → icon shows Vale logo, name shows "TaxiBook EPS"
  • Opening from home screen icon launches full-screen (no browser address bar)
  • Notification permission prompt appears on first open — user taps Allow
  • iOS only: Notification permission requested after add-to-home-screen (not just browser)
3
Staff Books a Trip
Staff
Have the Coordinator device open on the home page — you'll verify real-time updates appear without refreshing.
  • Login as Staff → lands on home page, shows booking list (empty is OK) Staff device
  • Tap Book a Trip → booking form opens Staff device
  • Fill in pickup, destination, date/time, passenger count — tap Submit Staff device
  • Booking appears on Staff home with status Submitted Staff device
  • Coordinator sees new booking in "Needs Approval" in real time — no refresh needed Coordinator device
  • Try booking with Saved Location as pickup or destination (if locations are configured) Staff device

Running the Trip (GPS + Map)
All roles
This is the most important session. Have all 3 devices open. The driver should be in a real vehicle or walking with the phone to generate GPS movement.
  • Driver taps "Start Trip" — status → On Trip Driver device
  • Staff home shows On Trip badge in real time — no refresh Staff device
  • Driver marker appears on Coordinator fleet map with taxi name label Coordinator device
  • Route line (road-following) appears from driver to destination on map Coordinator device
  • Driver GPS marker moves on map as driver moves Coordinator device
  • Fullscreen button on map works — browser goes fullscreen, Esc exits Coordinator device
  • Driver taps "Complete Trip" — status → Completed Driver device
  • Staff receives "Trip completed" push notification Staff device
  • Completed trip appears in Coordinator Calendar in gray with ✓ prefix and actual finish time Coordinator device
  • Completed trip appears in Dispatch Board calendar with gray block Open Dispatch Board → Day view → find the completed trip

7
Cron Auto-Complete & Reminders
Cron
Tip: To test without waiting — edit a booking's auto_complete_at in Supabase to 1 minute in the past, then wait for cron to fire (up to 5 min).
  • Auto-complete fires correctly A past-due booking gets status → Completed automatically, staff + driver receive push notification
  • Start-time reminder fires (T±2 min before trip) Staff gets "Your taxi is arriving now", driver gets "Time to pick up passenger"
  • Overdue reminder fires (trip not started, past scheduled time) Driver gets overdue alert, passenger gets delay notification after 5 min
  • Coordinator manual trigger works — tap cron button on home page → result dialog shows correct counts
  • Reminder is NOT sent twice — check that second cron run for the same booking skips it

9
Notifications Deep Check
All roles
For each item below, close the receiving app completely (swipe away) before triggering the notification — this tests background push delivery.
  • Push arrives when app is fully closed (not just minimized) Close app → trigger booking event → notification appears in system tray
  • Tapping a notification opens the correct page in the app
  • Notification bell icon on home page shows unread count badge
  • Notification list shows history of all received alerts
  • iOS: Push notification received on iPhone (app must be added to home screen first) iOS Safari 16.4+ required. Notification permission must be granted after adding to home screen.

Notification types to verify at least once:

  • Booking submitted (coord)
  • Booking confirmed (staff)
  • Booking rejected (staff)
  • Driver accepted (staff)
  • Trip started (staff)
  • Trip completed (staff + driver)
  • Overdue reminder (driver)
  • Auto-completed (staff + driver)

Edge Cases & Error Handling
Edge cases
  • Staff cancels before driver accepts → disappears from driver queue
  • Round Trip booking → two bookings created, both appear in calendar
  • Two bookings on same taxi at overlapping time → check how calendar/board displays them
  • Driver with no GPS → fleet map shows stale/gray marker, no route line drawn
  • Open app with no internet → error boundary shows friendly message (not blank white screen)
  • Wake lock: screen stays on while driver is on active trip page (Android)
  • iOS — no layout issues with notch / Dynamic Island on newer iPhones
  • Booking list loads more (pagination) when scrolling past 10 bookings
